## 1. プロダクト概要

**名前:** 京都 Local Explorer  
**目的:** 京都在住の地の利を活かし、旅行者・地元民に「地元民目線の京都体験」を提供するAIアシスタントWebアプリ  
**想定ユーザー:** 京都を訪れる旅行者、京都在住で新たな発見をしたい人  

---

## 2. ユーザーシナリオ（Use Cases）

<!-- 仕様駆動開発の核心。ここから実装仕様を導く -->

### UC-01: 天候に合わせたスポット提案
```
ユーザー: 「今日雨が降ってるんだけど、どこ行こう？」
期待する動作:
  - Google Search grounding で現在の天気を確認
  - 屋内スポット（寺社の拝観エリア, 美術館, 町家カフェ等）を提案
  - Google Maps リンク付きで最大3〜5件返す
  - 混雑状況も含めて案内する
```

### UC-02: 混雑回避ルートの提案
```
ユーザー: 「清水寺に行きたいけど混んでるのが嫌」
期待する動作:
  - 混雑しにくい時間帯を案内
  - 清水寺周辺の「穴場」スポットも合わせて提案（隠れ家データから優先）
  - アクセス方法（バス/電車）を案内
```

### UC-03: 地元民の隠れ家スポット検索
```
ユーザー: 「観光客が少ない本物の京都らしい場所教えて」
期待する動作:
  - RAGデータ（自作「京都隠れ家リスト.md」）を優先して返す
  - 有名観光地は提案しない
  - エリア・カテゴリ（食・寺社・自然等）で絞り込み可能
```

### UC-04: お気に入り保存
```
ユーザー: [ブックマークボタン押下]
期待する動作:
  - 提案されたスポット情報をFirestoreに保存
  - 次回ログイン時に「保存済みスポット」として表示
```

## 3. 機能仕様

### 3.1 AIエージェント
| 項目 | 仕様 |
|------|------|
| モデル | Vertex AI Agent Builder (Gemini 1.5 Flash) |
| Grounding | Google Search + Google Maps |
| RAGデータ | Cloud Storage → Vertex AI Vector Search |
| キャラクター | 京都在住10年目の地元民。丁寧だが少し京都弁 |
| Hallucination対策 | Very Low に設定 |

### 3.2 フロントエンド
| 項目 | 仕様 |
|------|------|
| フレームワーク | Streamlit |
| 入力 | テキスト + 現在地入力（オプション） |
| 出力 | テキスト回答 + Google Mapsリンク + ブックマークボタン |

### 3.3 データストア
| データ | 保存先 | 内容 |
|--------|--------|------|
| 隠れ家スポット | Cloud Storage → Vector Search | 手動作成Markdownから |
| 公式観光PDF | Cloud Storage → Vector Search | kyoto.travel等 |
| お気に入り | Firestore | ユーザー別ブックマーク |
